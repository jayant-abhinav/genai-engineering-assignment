## Problem 1 – Cost-Efficient RAG Application

Build a Retrieval-Augmented Generation (RAG) application using a low-cost vector database with evaluation of:

- Retrieval quality
- Answer quality
- Latency
- Infrastructure cost

## Problem 2 – LLM-as-Judge Evaluation Pipeline

Build an automated evaluation pipeline that uses a Large Language Model (LLM) to:

- Evaluate model responses
- Detect and mitigate judge bias
- Compare multiple model configurations
- Generate structured evaluation reports
